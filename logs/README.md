# Logs Directory

Automated logs from verification scripts and container sessions.

## Directory Structure

```
logs/
├── README.md                       (this file)
├── verify-host/                    Host system verification logs
│   ├── README.md                   Detailed guide for verification logs
│   └── verify-host-*.md            Timestamped verification reports
├── docker-builds/                  Docker image build logs
│   ├── README.md                   Detailed guide for build logs
│   └── build-*.md                  Timestamped build output
└── container-sessions/             Container terminal session logs
    ├── README.md                   Detailed guide for session logs
    └── session-*.md                Timestamped session recordings
```

## Log Types

### 1. Host Verification Logs (`verify-host/`)

**Generated by:** `./scripts/verify-host.sh`

**Purpose:** Verify host system configuration before running containers

**Contains:**
- X11 display and xhost configuration
- Docker installation and permissions
- NVIDIA GPU status and capabilities
- NVIDIA Docker runtime availability

**Quick view:**
```bash
cat $(ls -t logs/verify-host/*.md | head -1)
```

[**See detailed documentation →**](verify-host/README.md)

---

### 2. Docker Build Logs (`docker-builds/`)

**Generated by:** `./scripts/build.sh`

**Purpose:** Record the Docker image build process

**Contains:**
- Docker layer creation steps
- NVIDIA base image download progress
- Package installations (Edge, dependencies)
- Build timestamps and final status

**Quick view:**
```bash
cat $(ls -t logs/docker-builds/*.md | head -1)
```

[**See detailed documentation →**](docker-builds/README.md)

---

### 3. Container Session Logs (`container-sessions/`)

**Generated by:** `./scripts/run.sh`

**Purpose:** Record everything that happens inside the container

**Contains:**
- All commands typed in the container
- All command outputs and errors
- Container startup messages
- Session start/end timestamps

**Quick view:**
```bash
cat $(ls -t logs/container-sessions/*.md | head -1)
```

[**See detailed documentation →**](container-sessions/README.md)

---

## Quick Reference

### View Most Recent Logs

```bash
# Most recent verification log
cat $(ls -t logs/verify-host/*.md | head -1)

# Most recent build log
cat $(ls -t logs/docker-builds/*.md | head -1)

# Most recent session log
cat $(ls -t logs/container-sessions/*.md | head -1)
```

### Search Logs

```bash
# Search verification logs
grep -i "error" logs/verify-host/*.md

# Search build logs
grep -i "error\|failed" logs/docker-builds/*.md

# Search session logs
grep -i "nvidia" logs/container-sessions/*.md
```

### Cleanup Old Logs

```bash
# Delete logs older than 30 days
find logs -name "*.md" -mtime +30 -delete

# Keep only 20 most recent logs of each type
ls -t logs/verify-host/*.md | tail -n +21 | xargs rm -f
ls -t logs/docker-builds/*.md | tail -n +21 | xargs rm -f
ls -t logs/container-sessions/*.md | tail -n +21 | xargs rm -f
```

## Log Format

All logs are saved as **Markdown (.md)** files with:
- ✅ Clean formatting (no ANSI escape codes)
- ✅ Proper headers and structure
- ✅ Timestamps at start and end
- ✅ Readable in any text editor or markdown viewer

## Git Ignore

This directory is excluded from git tracking via `.gitignore` to prevent log accumulation in version control.
